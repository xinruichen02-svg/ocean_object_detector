# 项目目标

这个项目当前定位为学习型科研工程仓库，不是论文仓库。目标是通过逐步搭建工程结构、理解数据格式、运行小规模实验和记录失败案例，系统掌握水下目标检测项目的基本流程。

## 短期目标

建立仓库，学习目标检测基础，理解 YOLO 数据格式。

短期阶段重点是把项目结构搭起来，明确每类文件应该放在哪里，并理解目标检测任务的输入、输出、标注格式和评价指标。

## 中期目标

跑通水下数据集 baseline，记录训练和评估过程。

中期阶段重点是选取一个合适的数据集或小样例数据，完成从数据整理、配置文件、训练、验证到结果查看的完整流程，并把每一步遇到的问题记录下来。

## 长期目标

加入水下增强、视频跟踪、机器人感知接口。

长期阶段重点是从单张图像检测扩展到更接近水下机器人作业的工程场景，包括水下图像退化增强、视频稳定检测、目标跟踪，以及面向机器人系统的感知结果输出。

## 当前阶段边界

- 不写论文目标。
- 不假装已经完成实验。
- 不直接实现复杂系统。
- 不追求一开始就创新算法。
- 先把目标检测流程学清楚，再逐步扩展。
