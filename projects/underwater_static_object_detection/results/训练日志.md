| 项目 | 内容 |
| --- | --- |
| 实验名称 | 第一次原图训练 |
| 训练方式 | 直接训练（原始图像，不启用数据增强） |
| 任务 | DUO 水下目标检测 |
| 模型 | YOLO11n（`yolo11n.pt`，使用预训练权重） |

| 输入尺寸 | 640 |

| 随机种子 | 42 |
| 设备 | 单 GPU（`device: 0`） |
| 训练设置 | 确定性训练、AMP、矩形训练 |
| 训练耗时 | 14,588.2 秒（约 4 小时 3 分钟） |
| 完整结果 | [第一次直接训练](第一次直接训练/) |

### 结果

| 记录点 | Epoch | Precision | Recall | mAP50 | mAP50-95 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 最佳综合指标 | 23 | 0.78482 | 0.67584 | 0.74291 | 0.52387 |
| 最终轮次 | 80 | 0.82312 | 0.64644 | 0.71480 | 0.51138 |

最佳综合指标出现在第 23 轮。训练到第 80 轮后 Precision 上升，但 Recall、mAP50
和 mAP50-95 均低于第 23 轮。本次结果作为无数据增强的直接训练基线。

详细参数、逐轮指标、曲线和权重均在[第一次直接训练](第一次直接训练/)目录中。

### 结果分析

#### 1. 收敛与过拟合

- 训练集的 box、cls 和 DFL loss 持续下降，并都在第 80 轮达到最低值，说明模型
  仍在继续拟合训练数据。
- 验证集 cls loss 在第 23 轮最低（0.67969），验证集 DFL loss 在第 29 轮最低
  （0.95837），之后不再随训练 loss 同步改善。
- mAP50 和 mAP50-95 都在第 23 轮达到最高，分别为 0.74291 和 0.52387；第 80
  轮降至 0.71480 和 0.51138。Recall 也从最高的 0.68688（第 24 轮）降至
  0.64644。
- Precision 在后期仍有提升，最高为 0.84070（第 76 轮），但 Recall 和 mAP 没有
  同步上升。模型后期判断更保守，误报有所减少，但漏检增加。

综合来看，模型在约第 20 至 30 轮已经基本收敛，中后期出现轻度过拟合。实际使用
应优先选择 `weights/best.pt`，不应使用第 80 轮的 `last.pt` 代替最佳权重。

#### 2. 各类别表现

PR 曲线给出的各类别 AP50 为：

| 类别 | 中文 | AP50 | 归一化混淆矩阵对角值 |
| --- | --- | ---: | ---: |
| echinus | 海胆 | 0.889 | 0.87 |
| starfish | 海星 | 0.856 | 0.82 |
| holothurian | 海参 | 0.635 | 0.65 |
| scallop | 扇贝 | 0.590 | 0.56 |

海胆和海星是当前模型表现最好的两类。海参和扇贝明显较弱，尤其扇贝 AP50 最低，
且约 38% 的真实扇贝在该混淆矩阵设置下被归入背景；海参对应比例约为 34%。这说明
当前主要问题是海参、扇贝漏检，而不是四个目标类别之间的大量相互混淆。

在所有被误判为目标的背景样本中，海胆预测占 54%，海参和海星分别占 21% 和
20%。这里的数值表示背景误检的类别构成，并不等同于各类别的误检率；仍需结合
具体预测图片定位海胆误检常见的背景纹理。

#### 3. 当前结论

1. 本次直接训练已经得到可用基线，最佳 mAP50 为 0.74291，最佳 mAP50-95 为
   0.52387。
2. 80 轮训练偏长，最佳模型在第 23 轮已经出现；继续训练主要提高 Precision，未
   提高综合检测能力。
3. 下一阶段的主要改进对象应是海参和扇贝召回率，同时检查海胆的背景误检。
4. 仅凭本次训练不能确定薄弱类别是由样本数量、目标尺寸、遮挡还是水下画质造成，
   需要查看逐类别数据分布和失败样例后再下结论。

#### 4. 下一次实验建议

- 保持本次数据划分和随机种子不变，确保后续实验可与本基线直接对比。
- 将训练轮数先设为 40，并使用 early stopping；继续按验证集 mAP50-95 保存最佳
  权重，减少无收益训练时间。
- 优先导出海参、扇贝的漏检图片以及海胆的背景误检图片，按小目标、遮挡、模糊、
  偏色和复杂背景分类统计。
- 在完成失败案例统计后，再单独设计数据增强或提高输入分辨率的对照实验；每次只
  改一类因素，避免无法判断指标变化来源。
