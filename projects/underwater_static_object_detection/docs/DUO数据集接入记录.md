# DUO 数据集接入记录

## 当前决定

正式项目第一版采用 DUO，任务类型为四类目标检测，不是单标签图像分类。
DUO 面向水下机器人目标检测，一张图可以包含多个目标；保留边界框才能服务
后续目标定位、跟踪和机器人作业。

## 类别定义

| 项目类别编号 | 英文类别 | 中文含义 |
| --- | --- | --- |
| 0 | holothurian | 海参 |
| 1 | echinus | 海胆 |
| 2 | scallop | 扇贝 |
| 3 | starfish | 海星 |

这里的编号是转换为 YOLO 后的项目编号。转换时必须按原始 COCO 标注文件中
`categories` 的 `id` 和 `name` 建立映射，不能仅凭类别顺序猜测原始编号。

## 数据位置

- 原始 COCO 标注：`dataset/DUO/annotations/`
- 原训练图片：`dataset/DUO/images/train/`
- 原测试图片：`dataset/DUO/images/test/`
- YOLO 标签：`dataset/DUO/labels/train/` 和 `dataset/DUO/labels/test/`
- 数据划分清单：`dataset/DUO/splits/`
- YOLO 数据配置：`dataset/DUO/data.yaml`

数据已全部归入正式项目目录。验证集由原训练集按固定随机种子 `42` 划出 15%，
通过图片清单引用原文件，不复制、不压缩、不改变图像像素。

| train | 5670 | 训练 |

完整类别实例数和转换检查结果位于

跳过；越过图像边缘的框会裁剪到有效范围。

## 第一阶段边界

- 使用 DUO 原始图片训练四类 YOLO 检测 baseline。
- 不做颜色校正、去雾、亮度增强或生成增强图像。
- 关闭训练中的 HSV、翻转、几何变换、Mosaic、MixUp 等可选增强。
- 只保留模型读取所必需的尺寸适配和数值归一化。

## 接入后检查顺序

1. 查看 DUO 实际目录、图片数量和标注 JSON 文件名。
2. 读取 COCO `categories`，确认四个原始类别 ID。
3. 检查图片 ID、文件名、宽高和边界框是否完整。
4. 明确原始训练集、测试集以及验证集划分策略。
5. 将 COCO 边界框转换为 YOLO 归一化标签。
6. 随机可视化标签，确认类别和框位置正确。
7. 统计每类实例数量、空标签、越界框和缺失图片。

以上准备和自动检查已经完成。开始完整训练前仍应人工查看随机可视化样本。
